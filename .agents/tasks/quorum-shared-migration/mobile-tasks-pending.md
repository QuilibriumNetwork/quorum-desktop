# Mobile tasks pending — the unified tracker

> **This is the single tracker for every mobile-bound task**, regardless of where it originated: shared-package adoptions (this folder's work), desktop→mobile feature ports, and convergence work (see [../port-to-mobile/candidates.md](../port-to-mobile/candidates.md)). The `Category` column records the origin so provenance is never lost. It lives in the `quorum-shared-migration/` folder for historical reasons (many workflow docs point at this exact path), but its role is cross-cutting — do **not** create a second mobile-task tracker elsewhere; add rows here.
>
> Mobile's `.agents/` is gitignored, so mobile task files have no GitHub visibility. This table is the desktop-side bookkeeping: a list of mobile tasks dropped during our sessions, so we don't lose track of work that's been handed off.
>
> **Maintenance**: when a mobile task file moves to `.done/` (or is closed), update the row here. New rows are inserted in the row's correct urgency tier (NOT appended at the bottom — the table is sorted by urgency, then by drop date within a tier). Set the `Category` (`shared-migration` / `feature-port` / `convergence`) to match the task's origin; for a candidate graduating from [../port-to-mobile/candidates.md](../port-to-mobile/candidates.md), use the same value as its `Type` there. When you flip a row's urgency, re-sort and add a footer note explaining why.
>
> **Shared version note for executors**: task files written before today may pin older shared versions (e.g. `2.1.0-18`, `2.1.0-19`). Those are "available since" floors, not exact pins. When picking up any task, bump mobile's `@quilibrium/quorum-shared` to the **latest published version** (currently `2.1.0-28` as of 2026-06-10 — check npm for newer). Row 2 (profile-validation convergence) specifically **requires** `2.1.0-28`, which is where the Farcaster byte limits (`MAX_DISPLAY_NAME_BYTES`, `MAX_BIO_BYTES`) and the `.q` dot rule landed; `MAX_BIO_LENGTH` was removed in that release.

## Urgency tier legend

Tier reflects user-visible impact and irreversibility. Sequencing recommendation, not a deadline.

| Tier | Meaning |
|---|---|
| 🔴 Tier 1 | Real user-visible bug. Production users are affected today. Sequence first. |
| 🟠 Tier 2 | Silent correctness/security hole or cross-platform inconsistency with a real-world footprint. Sequence next. |
| 🟡 Tier 3 | Cleanup bundled with minor correctness wins (XSS defence-in-depth, timer leaks, masked bugs). Worth doing but not urgent. |
| 🟢 Tier 4 | Pure mechanical refactor, byte-for-byte equivalent, no runtime test required. Batch onto any unrelated PR. |
| 🚧 Blocked | Waiting on a lead-dev product call before scoping. |

## Category legend

Where the task originated. Provenance only — does not affect sequencing (urgency does).

| Category | Meaning |
|---|---|
| `shared-migration` | Mobile should adopt a `@quilibrium/quorum-shared` export (this folder's core work). The original and most common category. |
| `feature-port` | A desktop feature mobile lacks entirely. Graduated from [../port-to-mobile/candidates.md](../port-to-mobile/candidates.md) (`Type: feature-port`). |
| `convergence` | Mobile has the capability but desktop's implementation is better; mobile converges. Graduated from [../port-to-mobile/candidates.md](../port-to-mobile/candidates.md) (`Type: convergence`). |
