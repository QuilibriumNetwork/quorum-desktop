# Mobile tasks — where to find them

> **This file used to be a hand-maintained list of every mobile-bound task. That list went stale** — it lived here in **desktop**, but the work and its status changes happen in **mobile** sessions, so the desktop mirror drifted constantly. Even a "thin index" of all the rows still rots (rows ship, get reprioritized, get absorbed). So as of **2026-06-14** this is a **signpost, not a list**: it points at the live trackers in the repo where the work actually happens. There's nothing per-task to maintain here, so nothing to go stale.

## Where the mobile task queue lives (all in the mobile repo)

Mobile's `.agents/` is gitignored (local-only), so read these directly on disk — they have no GitHub visibility:

| You want… | Look here |
|---|---|
| **Shared-migration tasks** — live triage (which are doable now, which are blocked, symbol availability vs the installed dist) | `quorum-mobile/.agents/tasks/quorum-shared-migration/STATUS.md` |
| **Everything mobile** — the bird's-eye dashboard (all workstreams, what's next, blocked) | `quorum-mobile/.agents/RECAP.md` (regenerated by `/recap`) |
| **A specific task's full detail + current status** | the individual task file under `quorum-mobile/.agents/tasks/…` (the per-task source of truth) |
| **Feature-port / convergence candidates** not yet dropped as tasks | [`../port-to-mobile/candidates.md`](../port-to-mobile/candidates.md) |

## When you drop a new mobile task (from a desktop session)

1. Write the task file in the **mobile** repo (`quorum-mobile/.agents/tasks/…`, or `…/quorum-shared-migration/` for a shared-adoption task). That file carries the task's status — set its frontmatter `status` and keep it current there.
2. If it's a shared-migration task, add/triage it in mobile's `STATUS.md`.
3. Run `cd D:\GitHub\Quilibrium\quorum-mobile && python .agents/update-index.py` to refresh mobile's INDEX.
4. **Do NOT add a status row back here.** This file is a signpost. Recording the task in its mobile-side home (step 1-2) is what makes it visible to the next session — that's where they'll look, because this file sends them there.

> **If you ever want the at-a-glance cross-repo list back:** don't hand-maintain it (that's what rotted). Generate it — a script that scans mobile's task-file frontmatter + `STATUS.md` and rewrites a table from ground truth, the same way `INDEX.md` / `RECAP.md` are generated. Not built yet (signpost was enough); build it only if the glance is actually missed.
